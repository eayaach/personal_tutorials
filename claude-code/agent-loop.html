Core concept

The Agent Loop

I don't respond in a single pass. I loop — thinking, using tools, observing results — until the task is done.

💬User Input
→
🧠Think
→
🔧Use Tool
→
👁️Observe
→
✅Respond
or loop again
Phase 1

Think

🎯 Understand intent

Parse what you actually want, not just what you literally said.

🗂️ Check memory

Recall project context, conventions, and previous decisions.

📋 Plan steps

Break the task into a sequence of concrete actions before acting.

⚠️ Assess risk

Flag destructive or irreversible actions before executing them.

Phase 2

Use Tools

📁 Files

  • Read
  • Write
  • Edit
  • Glob / Grep

💻 Shell

  • Bash commands
  • Run tests
  • Install packages
  • Git operations

🌐 Web

  • WebSearch
  • WebFetch
  • Read docs
  • Check APIs

Each tool call result becomes input to the next thinking step.

Phase 3

Observe & Decide

✅ Success

Result matches expectations — move to the next step or respond to user.

❌ Error

Something failed — diagnose the cause, try a different approach.

🔄 Incomplete

More steps needed — loop back and continue executing.

🛑 Blocked

Cannot proceed safely or without missing info — ask the user.

Safety layer

Permission Model

Action type Reversible? Behavior
Read files, search code Yes Auto-approved, no prompt
Edit / write files Mostly Requires permission mode allow
Run shell commands Mostly Shown to user, can be denied
Git push, delete files, send messages No Always asks for confirmation
For you

How to work with the loop

🎯 Give goals, not steps

"Add auth to the app" beats "edit this file, then that file..."

🐛 Paste full errors

Stack traces and terminal output give me everything I need to debug.

✋ Deny risky tool calls

If you deny a tool call I'll stop and ask, never force through.

🔁 Redirect freely

If I go off track, just say so — I'll re-plan from where we are.
